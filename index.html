Semantic HTML reference
Tag Description
<header> Represents a container for introductory content or a set of navigational links. Typically contains the site logo, site title, and main navigation.
<nav> Represents a section of a page that contains navigation links.
<main> Represents the main content of a document. It should be unique to the document, excluding content that is repeated across a set of documents such as site headers, navigation links, and footers.
<article> Represents a self-contained composition in a document, such as a forum post, a magazine or newspaper story, or a blog entry.
<section> Represents a thematic grouping of content, such as chapters, headings, paragraphs.
<aside> Represents a section of a page that contains content that is indirectly related to the main content, and which could be considered separate from that content.
<footer> Represents a container for the footer of a document
<figure> Used to represent a self-contained unit of content, such as an image, illustration, or diagram, that is typically referenced as a single unit from the main flow of the document.
<figcaption> Used to provide a caption or a description for the content within a figure element. The figcaption element should be used as the first or last child of the figure element, and it can be used to provide a title, a subtitle, or a short explanation for the content within the figure.
<time> Represents a specific time or date. It can be used to provide a machine-readable format of a date or time, which can be used by search engines, browsers, and other software to understand the information on a webpage.
<mark> Highlights text within a document, typically to indicate a search term or to draw attention to a specific phrase or passage.
<details> Creates a widget that the user can open and close to reveal or hide additional content. The details element can be used to create an interactive element that allows users to reveal or hide additional information on a webpage, such as a summary, a list of related links, or a set of instructions.
<summary> Used in conjunction with the details element to provide a summary or a label for the additional content that can be revealed or hidden by the details element. The summary element should be the first child of the details element.
<picture> Represents a container for zero or more source elements and one
<h1>-<h6> Represents headings of different levels. The h1 element should be used for the main heading of a document, and the h2 - h6 elements can be used for subheadings.
<p> Represents a paragraph of text.
